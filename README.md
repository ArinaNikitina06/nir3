# НИР3

Прототип веб-интерфейса платформы курсов с персональными рекомендациями: Next.js (App Router), Prisma, NextAuth.

Макет в Figma: https://www.figma.com/design/XQiGCFwKH7yWHQwnbhJIoK/Untitled?node-id=0-1

Итоговый текст работы: [docs/NIR3-Nikitina-final.pdf](docs/NIR3-Nikitina-final.pdf).

## Где ссылка на сам сайт (HTTPS), а не на код

- **Страница проекта на GitHub** (код, коммиты): [github.com/ArinaNikitina06/nir3](https://github.com/ArinaNikitina06/nir3) — это не «сайт из макета», а хранилище исходников.
- Адрес с окончанием **`.git`** (`…/nir3.git`) нужен программе **git** (`git clone`), в браузере по нему открывается тот же GitHub.

**Работающее приложение в браузере** появляется только после **деплоя** (публикации) Next.js на хостинг. Тогда хостинг выдаст ссылку вида `https://имя-проекта.vercel.app` или другой домен.

Удобный вариант для Next.js — [Vercel](https://vercel.com): войти через GitHub → Add New Project → выбрать репозиторий `nir3` → в настройках проекта указать переменные окружения (`DATABASE_URL`, `NEXTAUTH_SECRET`, `NEXTAUTH_URL` — для продакшена в `NEXTAUTH_URL` укажите уже **URL сайта с Vercel**, например `https://nir3-xxx.vercel.app`). После деплоя Vercel покажет готовый HTTPS-адрес сайта.

Пока деплоя нет, интерфейс смотрят **на своём компьютере** (см. раздел «Запуск» ниже): после `npm run dev` откройте в браузере **http://127.0.0.1:3000** или **http://localhost:3000** (это не публичный адрес в интернете).

## Запуск (открыть сайт у себя)

1. Установите [Node.js](https://nodejs.org/) (LTS).
2. Склонируйте репозиторий и зайдите в папку проекта (или откройте папку `NIR` в редакторе кода).
3. В терминале **в папке проекта** (рядом с `package.json`):

```bash
npm install
npx prisma generate
```

4. Переменные окружения: скопируйте пример и при необходимости поправьте **`DATABASE_URL`**:

```bash
cp .env.example .env.local
```

5. **База данных (PostgreSQL).** Удобный вариант — Docker (нужен запущенный **Docker Desktop**):

```bash
npm run db:setup
```

Команда поднимет контейнер `nir3-postgres` на порту **5433** (порт **5432** часто уже занят другими проектами) и применит схему Prisma. Если Docker не запущен, сначала откройте Docker Desktop, затем снова выполните `npm run db:setup`.

Проверка, что БД жива: `docker compose ps` — статус `healthy`.

6. Применить схему вручную (если контейнер уже запущен):

```bash
npm run db:push
```

7. Запуск сервера разработки:

```bash
npm run dev
```

Если порт **3000** занят (другой Docker-проект), используйте `npm run dev:3001` и откройте **http://127.0.0.1:3001** (в `.env.local` задайте тот же адрес в `NEXTAUTH_URL`).

8. Дождитесь строки **«Ready»** в терминале. Откройте в **обычном браузере** (Chrome, Safari, Edge), а не на GitHub:

**http://127.0.0.1:3000** (или **:3001**, если запускали `dev:3001`)

Если страница не грузится: убедитесь, что терминал с `npm run dev` **не закрыт** и в нём нет красных ошибок; для регистрации — что **Docker Desktop запущен** и контейнер `nir3-postgres` в статусе `healthy` (`npm run db:up`).

Сборка: `npm run build`.

## Регистрация и вход

- **Регистрация:** `/auth/signup` — email, имя, пароль (не короче 8 символов), повтор пароля. Данные уходят в `POST /api/auth/signup`, пароль хранится как **bcrypt**-хеш в поле `User.passwordHash`.
- **Вход:** `/auth/signin` — NextAuth (Credentials). Пользователи **без** `passwordHash` (созданные старым скриптом) войти по паролю не смогут — зарегистрируйтесь заново или задайте пароль вручную в БД.

Дополнительные заметки и черновики лежат в каталоге `addinfo/`.
